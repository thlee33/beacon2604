# 조선시대 봉수(烽燧) 디지털 트윈 📡🔥

조선시대 **제2로 직봉(부산 응봉 ~ 한양 목멱산)** 통신 네트워크망을 3D 지형 위에 사실적으로 구현하고, 옛날 봉수군들처럼 봉수대 간의 **가시권(Line of Sight, LoS)**을 직접 시뮬레이션 및 검증해 볼 수 있는 3D 웹 디지털 트윈 프로젝트입니다.

---

## 🚀 프로젝트 개요
과거 국경의 위급한 소식을 중앙으로 전달하던 대표적 통신 수단인 '봉수'가 실제로 산맥의 방해 없이 눈으로 관측 가능했는지 체험할 수 있습니다. 
사용자가 좌측 UI에서 특정 봉수대(예: 계명산)를 선택하면, 카메라가 해당 지역의 산꼭대기(지표 30~100m 상공)로 날아가 **직전 봉수대**(예: 황령산) 방향을 정확하게 바라봅니다. 이때 직전 봉수대에서 피어오르는 **가상의 불꽃 파티클 이펙트**를 확인하며 두 거점 간 지형적 가림막이 없었음을 직관적으로 증명할 수 있습니다.

## 🛠 기술 스택
- **Language**: HTML5, CSS3, Vanilla JavaScript
- **3D WebGIS Engine**: [CesiumJS](https://cesium.com/platform/cesiumjs/) (v1.116)
  - Cesium World Terrain (고해상도 3D 지형)
  - Bing Maps Imagery (위성 영상 기본 베이스)
- **Visual Effects**: HTML5 2D Canvas를 활용한 경량화 불꽃 파티클 시스템

## ✨ 주요 기능 및 기술적 특징

1. **정밀한 3D 지형 트래킹 (LOD Parallax 극복)**
   - Cesium의 `HeightReference.RELATIVE_TO_GROUND` 및 `globe.getHeight()` API를 결합하여, 카메라가 멀고 가까워짐에 따라 다이나믹하게 변하는 지형 렌더링(LOD) 속에서도 불꽃과 지명이 미끄러지거나 붕 뜨지 않고 **산꼭대기 표면에 완벽하게 밀착(+15m 우위)**되도록 구현했습니다.
2. **2D Canvas 기반 동적 불꽃 엔진**
   - 무겁고 관리가 까다로운 WebGL 3D 파티클 대신 가벼운 2D HTML Canvas를 뷰어 위에 오버레이 했습니다.
   - `Cesium.SceneTransforms.wgs84ToWindowCoordinates`를 매 프레임(`postRender`)마다 계산하여, 3D 공간상의 봉수 점화 위치를 2D 화면 좌표로 투영해 리얼타임으로 파티클을 스폰함으로써 퍼포먼스와 시각적 아름다움을 동시에 확보했습니다.
3. **가시권(LoS) 스텝-바이-스텝 시뮬레이터**
   - 첫 번째 봉수부터 순차적으로 거슬러 올라가며 이전 봉수를 바라보는 카메라 방향 수치 연산 (Vector Math를 통한 `direction`, `up` 구성)을 적용했습니다.
   - 울퉁불퉁한 계명산, 내포점 등 카메라 시야가 확보되기 힘든 특이 지형을 위해, GeoJSON에서 개별 높이 위상(`height`) 속성으로 옵셋 오버라이딩을 지원하며 카메라 고도를 상황에 맞게 100m 상공까지 탄력적으로 띄울 수 있도록 설계했습니다.
4. **글래스모피즘(Glassmorphism) UX/UI 스타일**
   - 아름다운 디지털 트윈 배경을 시각적으로 방해하지 않기 위해 반투명 블러(Blur) 기반의 최신 글래스모피즘 테마를 입혔습니다. 아코디언 메뉴 형식으로 공간 활용도를 높였습니다.

## 📂 폴더 구조

```text
beacon2604/
├── docs/                   # 프로젝트 관련 문서 자료실 (기술 블로그, PRD 등)
└── web/
    ├── data/
    │   └── beacon.geojson  # 대동여지도 기반 제2로 봉수대 수급 좌표계 (공간 데이터)
    ├── img/
    │   └── fire.svg        # 초기 점화 기본 위치를 잡아주는 베이스 아이콘
    ├── index.html          # 메인 프레임 (사이드바 메뉴 및 Cesium, Canvas 겹침 구조)
    ├── style.css           # 글래스모피즘 CSS 스타일시트 
    └── main.js             # 데이터 비동기 파싱, 카메라 연동, 파티클 루프 로직 통합
```

## 💻 로컬 실행 방법
Node.js(또는 `npx`가 설치된 서버 환경)에서 다음 커맨드를 통해 프로젝트를 띄웁니다.

```bash
git clone https://github.com/thlee33/beacon2604.git
cd beacon2604/web
npx serve -l 4422
```
이후 `http://localhost:4422`에 접속하시면 시뮬레이션을 즉시 확인할 수 있습니다.
